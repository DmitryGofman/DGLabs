From vibe coding to real engineering

The Iceberg

Vibe coding is the tip. Engineering is everything beneath.

The shiny part — chatting with an AI and watching code appear — is real and powerful. But durable tools live below the waterline. Tap any layer to see what it means and why it matters.

Above the water · the easy 10%

Vibe Coding

Describe what you want in plain language and watch code appear. This is the genuine superpower — it collapses the time from idea to first draft. Tools like Claude Code make it conversational.
A script that runs once on your machine. Perfect for exploring an idea — but a demo is not yet a tool other people (or robots) can depend on.
Fast and satisfying. The trap is that you don't yet understand why it works — so when it breaks, you're stuck. This path closes that gap.
Below the water · the real 90%

Production Reality

Git, branches, pull requests. How you experiment safely, undo mistakes, and let an AI edit your files without fear. Covered in Track 01.
How you prove code works and keep it working — automatically, on every change. The difference between "ran once" and "reliable." See Track 03.
Docker, dependencies, lockfiles, servers, edge devices. Making your tool run the same way everywhere — including on real hardware.
Storing sensor streams, talking to other systems, validating inputs you don't control. Most engineering is, quietly, data handling.
Knowing what to build and which tradeoffs to make. The agent accelerates the typing; this path builds the judgement that directs it.

Good news: with Claude Code as your partner, you don't have to memorise the depths. You learn to recognise them, ask the right questions, and let the agent do the heavy lifting — under your direction.

Prompt Lab

Your output is only as good as your ask

The highest-leverage skill in AI-assisted engineering isn't typing — it's specifying. Write a prompt below and get instant feedback on what would make it sharper.

Grade my prompt

Write the instruction you'd give Claude Code, then check it against five marks of a strong prompt.

The same request, two ways

Notice how the strong version supplies context, constraints, and a definition of done.

✕ Vague "make a script to read my sensor"
✓ Specific "Write a Python script that reads my IMU over serial at 100 Hz, applies a low-pass filter, logs to CSV with timestamps, and handles disconnects gracefully. Use only pyserial + numpy. Show me a plan first."

Context · Constraints · A "done" condition · Plan-first. Master this and the agent feels twice as capable.

Knowledge check

Quick gut-check

Three questions on the mindset that turns an AI assistant into an engineering multiplier. Pick an answer to reveal the reasoning.

Question 01

You're about to ask Claude Code to add a non-trivial feature. What's the highest-leverage first move?

Planning before building is the core discipline that separates engineering from vibe coding. Agreeing on the approach while changes are still cheap prevents wasted work and surfaces risks early.
Question 02

Why keep a CLAUDE.md file in your project?

CLAUDE.md is durable memory. It means you don't re-explain your project every session, and the agent makes choices that match your conventions. Generate a first draft with /init.
Question 03

Your AI-written script worked once. What makes it a tool you can actually trust?

"Ran once" is the tip of the iceberg. Tests prove it keeps working, a declared environment makes it run anywhere, and version control lets you change it safely — that's the production reality below the waterline.
