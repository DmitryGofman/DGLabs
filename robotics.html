03 · Coding & AI Tools

The languages, and a force multiplier

Python and C/C++ are the most-used languages in robotics. Pair them with disciplined ROS2 development and real-time programming — and use Claude to generate boilerplate, tests, and documentation so you spend your time on the hard parts.

Python & C/C++

Python for high-level logic and ML; C/C++ for real-time and embedded control code.

ROS2 Development

Write nodes and services in C++ and Python; build with colcon; communicate over DDS middleware.

Real-Time Programming

FreeRTOS / RT Linux, multithreading, and deterministic timing for stable control loops.

Claude & AI Tools

Be explicit and structure instructions as steps. Use AI for boilerplate, test generation, and docs.

generate_node.py — Claude as a coding co-pilot 
from anthropic import Anthropic

client = Anthropic(api_key="YOUR_API_KEY")

# Prompt Claude to generate a ROS2 node for LiDAR obstacle detection
resp = client.messages.create(
    model="claude-sonnet-4-6",   # use a current, capable model
    max_tokens=1024,
    messages=[{"role": "user", "content":
        "Generate a Python ROS2 node that subscribes to '/scan' "
        "and publishes the nearest obstacle distance. Comment every line."}],
)
print(resp.content)

07 · Career & Interviews

From study plan to job offer

Target the roles your cross-disciplinary fit suits, build a portfolio that proves it, and prepare to walk an interviewer through your work.

Robotics Engineer

End-to-end design of robotic systems across software and hardware.

Automation Engineer

Manufacturing and logistics automation, PLCs, and integration.

Control Systems Engineer

Motion control, state estimation, and stability for dynamic systems.

Perception / AI Engineer

Vision, LiDAR, and learned models for autonomy.

Build a portfolio

  • Curate a GitHub with polished code and a README per project.
  • Include simulation demos and short video walk-throughs.
  • For each project, state your role, the challenges, and your solutions.
  • Keep an engineering decision log as you build (see Avirlog).

Prepare for interviews

  • Review fundamentals: kinematics, control, and core algorithms.
  • Practice problems: sensor calibration, writing ROS nodes, system design.
  • Use the STAR method to narrate projects and teamwork.
  • Expect a candidate-led walkthrough of your past work — rehearse it.
